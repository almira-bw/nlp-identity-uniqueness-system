01 Executive Summary

Pefindo's Productive Lending underwriting pipeline requires a systematic, auditable method for detecting identity attribute inconsistencies across multiple loan applications from the same borrower. Manual review of address, phone number, and ID variations is slow, unreliable, and produces no audit trail.

The NLP Identity Uniqueness System addresses this gap through three automated NLP modules — each measuring distinct value counts for a single identity attribute — and a configurable risk scoring layer that feeds directly into the underwriting scorecard. The system was developed under a rigorous 8-phase methodology with MLflow experiment tracking, test-driven development, and a locked test set evaluation protocol.

03 Technical Approach

The Address Module employs a three-layer ensemble that escalates to machine learning only when rule-based logic cannot make a confident determination. Phone and ID modules apply deterministic rule-based logic, with the best-performing variants selected via empirical validation.

01Layer
Rule-Based Normalization
Expands Indonesian address abbreviations (Jl.→Jalan, Kec.→Kecamatan, Kel.→Kelurahan). Strips stopwords and removes ALAMAT prefix artifacts. Normalizes RT/RW leading zeros to ensure consistent formatting before comparison.
Coverage: All address pairs
02Layer
Fuzzy + TF-IDF Feature Extraction
Generates 5 features per address pair: token_sort_ratio, token_set_ratio, partial_ratio, length_ratio, tfidf_cosine. Hard rules applied — Token Set ≥ 88 → SAME; Token Set < 78 → DIFFERENT. The borderline zone (78–87) escalates to Layer 3.
Handles ~majority of pairs with high confidence
03Layer
Trained ML Classifier
GradientBoostingClassifier selected from a 9-candidate sweep via MLflow tracking. Trained on 10,000 stratified address pairs covering 5,583 of 5,654 eligible identities (99%). 5-fold cross-validation F1 = 0.993. Validation holdout F1 = 1.000. Train–val overfitting gap = 0.007 (threshold: ≤ 0.10 ✓).
Borderline zone only — escalation from Layer 2

06 Quality Gates & Development Methodology

The system was built under a rigorous 8-phase development process with embedded guardrails at every stage. The methodology is designed to prevent data leakage, overfitting, and regression — ensuring production deployments are backed by an immutable audit record.

A
Test Set Lockout
test.csv was sealed until Phase 7. All model selection decisions used the validation set exclusively. Zero test data leakage by design.
✓ Zero leakage
B
Test-Driven Development
Every function required a failing test before implementation. Coverage: 95% preprocessing · 85% clustering · 100% risk categories. 127 tests passing.
✓ 127 tests pass
C
Overfitting Gate
Learning curve computed via 5-fold StratifiedKFold. Train–val F1 gap must be ≤ 0.10 before test set unlock.
✓ Gap = 0.007
D
Regression Baseline Lock
Phase 9 baselines locked in test_regression.py post-evaluation. Any future model change must pass 61 regression tests before deployment.
✓ 61 regression tests
E
Stratified Identity Split
Train/val/test split performed at identity level (70/15/15). No borrower appears across splits — prevents identity-level data leakage.
✓ No cross-split leakage
F
MLflow Experiment Tracking
All 9 classifier candidates, 21 address variants, and 15 phone variants logged with precision, recall, F1. Final evaluation logged as FINAL-TEST-EVAL — immutable audit record.
✓ Full audit trail
07 Known Limitations

The following limitations are acknowledged and have been mitigated for Phase 1 deployment. Each is in scope for resolution in subsequent development phases.

Area Limitation Mitigation Applied
Address label quality Classifier accuracy depends on auto-label quality in training data. 3-signal rule validated; dual annotation with kappa check on 100 pairs planned.
Phone prefix grouping Prefix match may group different subscribers sharing a number prefix. Levenshtein layer added; Prefix@9 selected empirically on validation set.
KTP/NPWP rule-based only No ML component for ID matching in Phase 1. Structural validation sufficient for Phase 1; ML layer scoped for Phase 2.
